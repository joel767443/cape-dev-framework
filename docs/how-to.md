# How to (backend developer guide)

This guide is for developers extending the backend framework in this repo.

## Installation

From the repo root:

```bash
composer install
php -S localhost:8001 -t public public/index.php
```

Or start the backend dev server:

```bash
php run dev
```

### Common environment variables

- `APP_DEBUG`: `1|true|yes|on` enables stack traces in JSON errors
- `DB_SQLITE_PATH`: override SQLite file path
- `JWT_SECRET`: enable JWT issuing/verifying (`/api/auth/token`)

## Routing

### Where routes live

Routes are defined in `routes/api.php`.

### Register a route

Use `WebApp\Router` (see `src/Router.php`):

- `get($path, [$controllerClass, $method])`
- `post($path, [$controllerClass, $method])`
- `add($method, $path, [$controllerClass, $method], $opts)`

Example:

```php
$router->get('/api/health', [App\Http\Controllers\HealthController::class, 'show']);
```

### Route params + constraints

Use Symfony-style parameters in the path and constrain them with `where`:

```php
$router->add('GET', '/api/users/{id}', [UserController::class, 'show'], [
  'where' => ['id' => '\\d+'],
]);
```

### Route middleware

Attach middleware via `middleware` (aliases come from `MiddlewareRegistry`):

```php
$router->add('GET', '/api/secure/ping', [AuthController::class, 'ping'], [
  'middleware' => ['auth_jwt'],
]);
```

### List routes

```bash
php run route:list
```

## Controllers

### Where controllers live

Framework controllers in this repo live under:

- `src/Http/Controllers/*`

Generated app controllers (via CLI) live under:

- `app/Http/Controllers/*`

### Return type

Controller actions **must return** a `Symfony\Component\HttpFoundation\Response` (usually `JsonResponse`).

### Argument injection

The kernel (`src/Http/Kernel.php`) can inject:

- `Symfony\Component\HttpFoundation\Request`
- any DI-resolvable service (via PHP-DI)
- `App\Requests\FormRequest` subclasses (auto-validated before your controller runs)

## Models

### Eloquent (default)

Base model:

- `app/Models/Model.php` (extends Eloquent)

Generate a model:

```bash
php run make:model User
```

### UUID primary keys (optional)

Use:

- `App\Models\Concerns\UsesUuid`

This switches the model to a string PK, non-incrementing, and assigns a UUID on `creating`.

### Migrations

```bash
php run make:migration create_users_table
php run migrate
```

### Doctrine ORM (optional)

Entities live in:

- `app/Entities`

Schema update:

```bash
php run doctrine:schema:update --dump-sql
php run doctrine:schema:update --force
```

## Requests + validation

### FormRequest

Base request:

- `app/Http/Requests/FormRequest.php`
- (new structure) `app/Requests/FormRequest.php`

Generate a request:

```bash
php run make:request StoreUserRequest
```

Add Symfony Validator constraints in `constraints()`.

### Validation errors

Validation failures return **422** JSON errors generated by `WebApp\Validation\RequestValidator`.

Example request:

- `app/Http/Requests/ValidateExampleRequest.php`
- Route: `POST /api/validate`
